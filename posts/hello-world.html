Hello World

March 21, 2026 · Marcus Vorwaller

Every blog starts with a hello world. This one is no different, except maybe in spirit. This isn't a corporate blog with a content calendar and SEO strategy. It's a place to think out loud about the things I'm building, the problems I'm solving, and the ideas that won't leave me alone at 2am.

I've been building software for a long time. Long enough to know that the best ideas usually come from the intersection of doing the work and writing about the work. They feed each other. So here we are.

What to expect

This blog will cover whatever I'm thinking about. Loosely, that means:

  • Systems design and architecture decisions — the kind that keep you up at night
  • Tools, workflows, and the craft of building software well
  • Experiments and side projects from the lab
  • Occasional opinions, offered with appropriate humility

I'll aim for substance over frequency. No filler posts. If I don't have something worth saying, I'll stay quiet.

The stack

This blog is deliberately simple. Static HTML, a single CSS file, no build step. Posts are just HTML files. To publish, I write markdown, convert it, and push. That's it.

Here's roughly what the workflow looks like:

# Write a new post
vim drafts/new-post.md

# Convert and publish
cp posts/template.html posts/new-post.html
# paste in the converted content, update index
git add -A && git commit -m "new post: the title"
git push

No static site generator, no dependencies to update, no build pipeline to break. The fastest build system is no build system.

Simplicity is the ultimate sophistication. I don't know if Leonardo da Vinci actually said that, but whoever did was right.

A small example

Since this is a technical blog, let's make sure code looks good. Here's a function I keep in my toolbox — a simple debounce:

function debounce(fn, delay) {
  let timer;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// Usage
const handleResize = debounce(() => {
  console.log('Window resized');
  recalculateLayout();
}, 250);

Nothing fancy. Just clear, readable code on a dark background that doesn't strain your eyes. That's all you really need.
